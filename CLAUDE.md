# 校閲・校正の手順と規約

このリポジトリでは `/proofread <ファイルパス>` または自然言語で依頼することで、
MarkdownファイルのAI校閲・校正を行います。

## 作業フロー

校閲・校正は **2フェーズ** に分けて行います。

### フェーズ1: 校閲

内容面の改善を提案します。具体的には以下を確認します。

- より適切・明確な表現がある箇所
- 説明が不足しており補足できる箇所
- 事実関係の誤りや疑わしい記述
- 著者の意図に沿わないと思われる記述
- 構成上の問題

各指摘は以下の形式で列挙します。

```
【校閲】
- 行番号（またはキーワード）: 指摘内容
  提案: 変更案（または削除・補足を推奨する旨）
```

提案を提示した後、ユーザーの承認を得てから修正を適用します。
承認された変更のみを反映した中間ファイルを作成します。

### フェーズ2: 校正

下記の「校正規約」に基づき、誤植・体裁の不備を修正します。
内容への踏み込み（校閲）は行いません。

各指摘は以下の形式で列挙します。

```
【校正】
- 行番号（またはキーワード）: 指摘内容
  修正前: 該当箇所
  修正後: 修正案
```

指摘を提示した後、ユーザーの承認を得てから修正を適用します。

### 出力

最終的な校正済みファイルを `results/<元のファイル名>` に保存します。
変更前後の差分（diff）も表示します。

# 校正規約（校正フェーズで必ず参照すること）

## 校正と校閲の区別

- **校正**: 誤植・体裁上の不備を修正する（本規約の対象）
- **校閲**: 文章の内容・意見に踏み込む修正（フェーズ1で実施済み）

校正フェーズでは校閲を行わないこと。

## 括弧と句点の位置関係

文章内で括弧を用いる場合、以下の形式のみを用いること。

**正しい形式（4番）:**
```
Twitter上に幼女はいない（これはわたしが知って最も落胆した事実だった）。
```

1. `〜。（〜。）` → 不可
2. `〜（〜。）` → 不可
3. `〜。（〜）` → 不可
4. `〜（〜）。` → 正しい

## 括弧の種類

- 中身が**英字のみ**の場合: 半角括弧 `( )` を使用
- それ以外: 全角括弧 `（ ）` を使用

## 参考文献の表記

本文中での参照:
- 和書: 二重鍵括弧 `『書名』`
- 英書: イタリック体 `*Book Name*`

脚注等での書籍情報に含める項目: 著者、訳者、出版社、発行年、ISBNコード

区切り記号:
| 種類 | 和書 | 英書 |
|------|------|------|
| 項目間 | 読点 `、` | 欧文ピリオド+スペース `. ` |
| 著者間 | 読点 `、` | 欧文カンマ+スペース `, ` |
| ミドルネーム | 全角中点 `・` | 半角スペース |

## 日本語と英数字の間のスペース

和文中に英単語・英数字が出現する場合、その前後に半角スペースを入れること。

```
誤: 私はclaudeを使う
正: 私は claude を使う

誤: Python3.12がリリースされた
正: Python 3.12 がリリースされた
```

ただし、以下の場合は除く。

- 句読点・括弧類が隣接する場合（スペース不要）
  - 例: `（claude）` → スペースなし、`、claude` → スペースなし
- URLやファイルパスなど、スペースを入れると意味が変わる場合

## 句読点・カンマ・ピリオド

各記事内で統一すること（記事間での統一は不要）。

カンマ・ピリオドを使用する場合:
- 英文: 欧文カンマ・欧文ピリオドの後に欧文スペースを入れる
- 和文: 全角カンマ `，` または全角ピリオド `．` を使用
- この場合、参考文献の読点 `、` は適宜全角カンマ `，` に読み替える

## 感嘆符・疑問符

1つの場合:
- 全角 `！` `？` を使用
- 句点として使われており、かつ後ろが閉じ括弧類でない場合 → 後ろに二分スペース（半角スペース）を入れる
- かぎ括弧内の最後などでは空白を入れない

2つの場合:
- `‼` を使用

## 三点リーダ

倍角（2つ続けて）使用すること: `……`

## ハイフン・ダッシュ

| 種類 | 用途 | 表記 |
|------|------|------|
| ハイフン | 複合語 | `built-in` |
| enダッシュ | ページ範囲等 | `pp.123--456` |
| emダッシュ | 欧文中のダッシュ | `---` |
| 二倍ダーシ | 和文中のダッシュ | `――` |

## 脚注番号

句点の**前**に置くこと。

```
正: 〜立ち向かうことは難しかった[^1]。
誤: 〜立ち向かうことは難しかった。[^1]
```

## 数式

| 誤 | 正 |
|----|-----|
| `$\Gamma, M, \tau$` | `$\Gamma$, $M$, $\tau$` （複数の変数はそれぞれ独立した数式に）|
| `$sin(x)$` | `$\sin(x)$` |
| `$\int f dx$` | `$\int f \mathrm{d}x$` |

## 強制改行

Markdown の強制改行（行末の半角スペース2つ以上、または `<br>`）は極力使わないこと。
